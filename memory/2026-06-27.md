## 27.06.2026 — Post-mortem: сбой консилиума моделей

### Сценарий сбоя
- Задача: запустить консилиум из 3 моделей (GLM 5.2, DeepSeek V4 Pro, Claude Opus 4.8)
- Исполнитель: sessions_spawn → sub-agent'ы → каждый дёргает OpenRouter
- Проблема: API-ключ OpenRouter закончился (insufficient balance / 402)
- Симптом: sub-agent'ы не вернули результат, sessions_yield завис, новая сессия не могла стартануть
- 3 последующие попытки пользователя проверить статус упали в те же грабли (ключ пуст)

### Причины
- Тройной расход токенов за раз на дорогих моделях
- Отсутствие проверки баланса перед вызовом
- Отсутствие fallback-стратегии
- sessions_yield без таймаута

### Уроки и фиксы
1. [rule] Дорогие модели — НЕ в sub-agent рои. Только прямые задачи.
2. [rule] Перед спавном sub-agent'ов — проверять лимиты API.
3. [fix] Добавить fallback-цепочку: дорогая → дешёвая той же семьи → Flash.
4. [fix] sessions_yield — с таймаутом и обработкой ошибок.
5. [rule] Если API упал — не пытаться повторить 3 раза с теми же моделями.
6. [todo] Обсудить с Дмитрием: как сделать консилиум без риска — через ротацию или кэш 1 ответа.

### Рекомендуемая архитектура консилиума (revised)
- Вместо 3 одновременных sub-agent'ов: последовательный опрос
- Flash → если ОК → Pro на критических вопросах → только если нужно → GLM/Claude
- Или OmniRoute с auto-fallback между провайдерами
- Не spawn-ить дорогие модели, а проксировать через единый gateway
