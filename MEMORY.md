## Фреймворк выбора моделей (Владимир Иванов)

### Почему каждая модель сильна в своём
- Claude: мощнейший SFT на огромных датасетах — «знающий», помнит кейсы из обучения. Всего 3% персонала Anthropic заняты RL.
- Gemini: сильнейшая американская LLM в RL (классическое PPO) — «понимающий». Очень быстрая.
- DeepSeek: сильнейшая LLM в GRPO-обучении — «оптимизатор». Чемпион по дешевизне. Гиперконнекции = не впадает в зомби-режим.

### Когда что использовать
1. Claude — «когда нужно вспомнить хороший пример»
   - Фронтенд/дизайн: описать все элементы UI сложно → богатый SFT спасает
   - Спецификации и прототипы UI: минимальный расход токенов
   - Бэкенд для новичков: знания Claude спасут
   - Бэкенд для профи: не так силён (инновации не в SFT)
2. Gemini — «король промптов и инновационных решений»
   - RL позволяет усваивать нестандартные концепции
   - Создание промптов для тонких воздействий
   - Подбор новых технических концепций
3. DeepSeek — «король оптимизации»
   - GRPO = эволюция алгоритмов с естественным отбором
   - Правило-based метрики (CPU, память, диск) — нечеловеческое программирование
   - Скорость бэкенда, точность реализаций
   - Во фронтенде слаб → нужен пример дизайна от Claude (даже в виде кода)
   - Лучший worker-агент (не впадает в зомби-режим)

## Agent Hooks (обновлено 2026-06-27)

### Behavioural hooks (конституция + правила)
Эти hooks я соблюдаю как агент, без внешних скриптов:

1. **preflight-check** — перед дорогими операциями (консилиум, spawn sub-agent'ов, GLM/Claude) запускать `~/.openclaw/preflight.sh MODEL`
2. **balance-check** — если баланс OpenRouter < $5 — не использовать Pro, < $10 — не использовать GLM/Claude
3. **expensive-no-spawn** — дорогие модели НЕ в sub-agent рои, только прямые задачи
4. **fallback-chain** — если дорогая модель недоступна → Flash
5. **yield-timeout** — sessions_yield только с таймаутом и обработкой ошибок
6. **no-retry-same** — если API упал, не повторять 3× той же моделью

### Файловые hooks (IIA bot)
- `~/.iia/agent-hooks/preflight-balance.mjs` — добавляет tool `check_api_balance` для бота
- Pre-flight скрипт: `~/.openclaw/preflight.sh`
