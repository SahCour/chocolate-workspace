## 1. Что такое S3-совместимое хранилище?

**S3 (Simple Storage Service)** — это стандарт облачного объектного хранилища, изобретённый Amazon Web Services. Он стал де-факто индустриальным стандартом благодаря простому HTTP(S) API.

**S3-совместимое** значит, что провайдер реализует тот же API, что и Amazon S3. Это даёт:

- **Единый интерфейс** — любой S3-клиент (boto3, rclone, mc, aws-cli) работает с любым S3-провайдером
- **Взаимозаменяемость** — можно переехать с одного провайдера на другой без переписывания кода
- **Экосистема** — тысячи инструментов, библиотек, CDN, backup-решений заточены под S3 API

**Ключевые концепции:**
- **Bucket** — корзина (контейнер для объектов)
- **Object** — файл + метаданные + уникальный ID
- **Key** — путь к объекту внутри bucket
- **Region** — географический дата-центр
- **Endpoint** — URL для доступа к API

## 2. Какие параметры хранилища важны для нас

### Критические

| Параметр | Почему важен |
|---|---|
| **S3-совместимость** | Чтобы rclone, утилиты бэкапа и код работали без переделок |
| **Надёжность** | Репликация, durability (99.999999999%), защита от потери данных |
| **Egress fees** | Сколько стоят скачивания — может быть дороже самого хранения |
| **API requests cost** | Цена за PUT/GET/LIST — критично для backup-операций |
| **Регион** | Желательно Европа/Азия — ближе к нашему серверу (Нидерланды — DO) |

### Важные

| Параметр | Почему важен |
|---|---|
| **Object Lock / Immutability** | Защита от ransom-шифрования и случайного удаления |
| **Versioning** | Возможность откатить случайно изменённый файл |
| **SLA / Uptime** | 99.9%+ |
| **TTL / Lifecycle** | Автоматическое удаление старых бэкапов |
| **Encryption at rest** | Шифрование на стороне провайдера |
| **Notification / Event hooks** | Для автоматизации (триггеры при загрузке файлов) |

### Приятно иметь

| Параметр | Комментарий |
|---|---|
| CDN интеграция | Для публичного доступа к контенту |
| Web UI | Удобно для ручного просмотра |
| CLI / SDK | Для скриптов и автоматизации |

## 3. Сравнение провайдеров

### Бюджетный сегмент (~$30-80/год за 1TB)

| Провайдер | Стоимость 1TB/год | Egress | S3-совм. | Комментарий |
|---|---|---|---|---|
| **Яндекс Диск** | ~$15-30 (500-990฿) | бесплатно | НЕТ | Не S3. Хорош как файловое хранилище, но не для инфраструктуры |
| **Backblaze B2** | ~$83 (6.95$/мес × 12) | бесплатно до 3× объёма | ✅ | Лучшее соотношение для нас. S3, минимум доп.плат |
| **Cloudflare R2** | от $0.015/GB/мес = $180/год | **бесплатно полностью** | ✅ | Нулевой egress. 10GB бесплатно. Нестандартный endpoint |
| **IDrive e2** | от $0.004/GB/мес = ~$48/год | бесплатно | ✅ | Очень дёшево. Есть yearly plans |
| **Wasabi** | ~$96 (7.99$/TB/мес резерв) | **бесплатно** | ✅ | С 1 июля 2026 — $7.99. Нет запроса egress. 1TB minimum |

### Средний сегмент

| Провайдер | Стоимость 1TB/год | Egress | S3-совм. | Комментарий |
|---|---|---|---|---|
| **Hetzner Object Storage** | ~€5/TB/мес = ~$60-70/год | €0.01/GB | ✅ | Мы уже на Hetzner? Если да — идеально. Германия, GDPR |
| **DigitalOcean Spaces** | $5/мес за 250GB (20$/TB) | **бесплатно** внутри DO сети | ✅ | Совпадает с нашим хостингом — нулевой egress |
| **Wasabi Reserved** | ~$60-70/год (1yr reserved) | бесплатно | ✅ | Выгоднее с контрактом |

### Enterprise / Гиганты

| Провайдер | Цена | Комментарий |
|---|---|---|
| **AWS S3** | ~$23/TB/мес + egress | Дорого для нас |
| **Google Cloud Storage** | ~$20/TB/мес + egress | Дорого |
| **Azure Blob** | ~$18/TB/мес + egress | Дорого |

## 4. Что подходит под нашу инфраструктуру

### Наш стек сейчас:
- Сервер: **DigitalOcean** (188.166.0.168, Нидерланды)
- AI/LLM: **OpenRouter** (внешний API)
- Деплой: **Railway** (опционально)
- Для хранения: ничего пока нет (77GB на VPS, из них 11GB занято)

### Лучшие варианты (в порядке приоритета):

#### 🥇 DigitalOcean Spaces
- **Совпадение с хостингом** — сервер и хранилище в одной сети (Амстердам), нулевой egress
- **$5/мес за 250GB**, пей-as-ю-гоу — можно начать с малого
- **S3-совместимое** — rclone, aws-cli, всё работает
- **CDN** — встроенный CDN для публичного контента
- Итого: **$60/год за 250GB**, или ~$240/год за 1TB

#### 🥈 Backblaze B2
- **$83/год за 1TB** — дешевле DO Spaces для больших объёмов
- **Бесплатный egress до 3× хранимого объёма** — для бэкапов хватит
- **S3-совместимое** — работает с rclone
- Запасной вариант для архивов и бэкапов

#### 🥉 Hetzner Object Storage (если переедем на Hetzner)
- **~€5/TB/мес** = очень дёшево
- Германия/NBG — GDPR-compliant
- S3-совместимое
- Актуально только если сервер переедет на Hetzner

#### 💡 Wildcard: Cloudflare R2
- **Бесплатный egress** — уникальное преимущество
- Интеграция с Cloudflare Workers
- **$0.015/GB/мес** (~$180/год за 1TB)
- Отлично для публичного контента и AI-данных

## 5. Рекомендуемая стратегия

### Сейчас (Phase 1)
1. **DigitalOcean Spaces** — основной бакет для инфраструктуры
   - Синхронизация workspace через rclone
   - Бэкапы конфигов OpenClaw
   - Хранение медиа-файлов

### После налаживания (Phase 2)
2. **Backblaze B2** — холодный бэкап
   - Ежедневная репликация DO Spaces → B2
   - Object Lock для защиты от ransom
   - Долгосрочные архивы логов

### Если переезд на Hetzner (Phase 3)
3. **Hetzner Object Storage** — замена DO Spaces
   - Внутренняя сеть Hetzner — нулевой egress
   - Единый провайдер для compute + storage
